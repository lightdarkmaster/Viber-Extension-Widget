Setting Up Viber Notifications
Automate Viber messages via your CRM workflow rules
1
Create a Message Template
Go to Devtac Message Templates in your CRM and create a new Viber template. You can personalize messages using dynamic placeholders — special tags that automatically pull real data from the CRM record when the message is sent.
How Placeholders Work

Wrap any CRM field's API name inside ${ } to insert its value automatically. The API name is the internal identifier Zoho uses, it may look slightly different from the label shown on screen.

Template:Hello ${First_Name}, your deal worth ${Amount} has been updated.
Sent as:Hello Juan, your deal worth ₱50,000 has been updated.
Types of Dynamic Values

a. Regular Fields

Any standard field on the record. Use the field's API name directly inside ${ }.

${First_Name} · First name of the contact
${Phone} · Contact's phone number
${Deal_Stage} · Current stage of the deal

b. Name or Address Sub-fields

Some fields like Full Name or Address contain multiple parts. Target a specific part using a dot after the field's API name.

${Full_Name.first_name} · First name portion
${Full_Name.last_name} · Last name portion
${Mailing_Address.city} · City from mailing address

c. Lookup Fields

Some fields on a record link to another record in CRM (e.g. a Deal has an Account Name field that links to an Account). You can pull values from that linked record using the lookup field's API name followed by a dot and the field you want.

${Account_Name.Phone} · Phone from linked account
${Contact_Name.Email} · Email from linked contact

d. Current Date Values

These are built-in placeholders that insert today's date information automatically, no field setup needed.

${System.Current_Day} · Current day (e.g. 31)
${System.Current_Month} · Current month (e.g. March)
${System.Current_Year} · Current year (e.g. 2026)

e. Record ID

Inserts the unique ID of the CRM record. Useful for building links.

${Record_Id} · Unique ID of the record
Finding a Field's API Name

Dynamic values use the field's API name, not its display label. To find it, go to:
Gear Icon (Top Right) → Developer Hub → APIs and SDKs → API Names tab, select your module and look for the API Name column next to each field.

Field LabelAPI Name to Use
First NameFirst_Name
Account NameAccount_Name
Closing DateClosing_Date

⚠️ Tip

If a placeholder has a typo or the field has no value on that record, it will either fail to resolve or send as blank text. Always double-check API names and test with a real record before activating your workflow.

2
Set Up a Workflow Rule
In Zoho CRM → Settings → Workflow Rules, create a rule on any module (e.g. Leads, Contacts, Deals). Set your trigger condition, such as a field update, record creation, or a scheduled date.
3
Add the Devtac Messaging Custom Action
In your workflow rule's Actions section, add a Custom Action and select Devtac Messaging. Choose your template, the phone field to send to, and the channel (Viber).
4
Viber Sends Automatically
Once the workflow rule triggers, the Viber is sent automatically — no manual action needed. You can monitor delivery logs in the Devtac Messaging Logs module in your CRM.

Showing the Chat Widget
Add the messaging panel to your CRM record pages
1
Open a CRM Record
Navigate to any record in the Leads, Contacts, or Deals module where you'd like the chat widget to appear.
2
Access the Related List Settings
In the top-left section of the record page, hover your cursor over Related List to reveal the settings option.
3
Open the Configuration Pop-up
Click Organize Potential Details to open the panel that controls which related lists are visible on the record.
4
Enable Devtac Viber Messaging
From the Unselected List, locate Devtac Viber Messaging and move it over to the Selected List.
5
Save and View the Widget
Click Save. The Devtac Viber Messaging widget will now appear on the record — ready for you to send messages directly from your CRM.
